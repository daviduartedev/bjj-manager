# Future Roadmap

## Context
Existe um conjunto de features pedidas que **não entram no MVP** mas
queremos manter visíveis para evitar perda de contexto e priorizar
melhor depois. Este ciclo é apenas documentação , nada é implementado.

## Intent
- Documento `docs/roadmap/future.md` com cada item:
  - resumo curto,
  - valor para o professor,
  - dependências técnicas conhecidas,
  - tamanho estimado (S/M/L).
- Itens mínimos a documentar:
  - Presença e frequência de alunos.
  - Turmas (organização por horário/turma).
  - Check-in via QR Code.
  - Integração com WhatsApp (lembretes de mensalidade).
  - Documentos do aluno (contratos, atestados).
  - Certificados de graduação (PDF imprimível).
  - Exportação CSV/Excel de alunos e financeiro.
  - Login de aluno e app do aluno.
  - OAuth providers (Google) no signup.
  - Convite de professores auxiliares na mesma conta (multi-user).
  - Roles (owner/coach/admin) com permissões granulares.
  - Dashboard avançado com gráficos e séries históricas.

## Taste / Constraints
- Apenas markdown.
- Não criar nenhuma tabela, página ou Server Action.
- Linguagem objetiva; nada de vendas.
